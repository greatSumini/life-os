# fitness-recommend Skill

최근 운동 기록과 목표/취약점을 분석하여 오늘의 운동 루틴을 추천하는 스킬이다.

## 파일 경로

- 운동 사전: `fitness/exercises.toml`
- 운동 로그 디렉터리: `fitness/logs/`
- 사용자 프로필: `fitness/profile.toml`
- 트레이너 리포트 디렉터리: `fitness/reports/`
- 분석 기준: `fitness/CLAUDE.md`

## 실행 절차

### Step 0. 현재 한국 날짜 확인

```bash
TZ=Asia/Seoul date '+%Y-%m-%d %A'
```

요일을 함께 확인하여 주간 패턴에 참고한다.

### Step 1. 데이터 수집

다음을 병렬로 읽는다:

**1-1. 최근 7일 운동 로그**

```bash
ls fitness/logs/ | sort -r | head -7
```

각 파일을 Read 툴로 읽는다.

**1-2. 운동 사전**

`fitness/exercises.toml`을 Read 툴로 읽는다.

**1-3. 사용자 프로필**

`fitness/profile.toml`을 Read 툴로 읽어 목표, 취약점, 근육 그룹 분류를 확인한다.

**1-4. 최근 트레이너 리포트**

```bash
ls fitness/reports/ | sort -r | head -1
```

가장 최근 리포트를 Read 툴로 읽는다 (있는 경우).

### Step 2. 주간 볼륨 분석

운동 로그의 exercise id를 `exercises.toml`과 조인하여 근육 그룹별 주간 세트 수를 집계한다.

```
근육 그룹별 주간 볼륨 (최근 7일):
- back: 18세트 (MAV 근접 ✓)
- chest: 6세트 (MEV 미달 ✗)
- shoulder: 8세트 (MEV 미달 ✗)
- legs: 12세트 (MEV 충족 ✓)
- arms: 4세트 (MEV 미달 ✗)
- core: 0세트 (MEV 미달 ✗)
```

기준:
- MEV (최소 유효 볼륨): 주당 10세트
- MAV (최대 적응 볼륨): 주당 20세트
- 목표 근육(등)은 MAV 근처, 나머지는 MEV 이상

### Step 3. 추천 근육 그룹 결정

사용자가 선호 부위를 지정했으면 해당 부위를 우선한다.
지정하지 않았으면 다음 우선순위로 결정한다:

1. **취약점 근육** (profile.toml의 weaknesses.areas) 중 볼륨 부족한 그룹
2. **목표 근육** (profile.toml의 goal.target_muscles) 중 MAV에 미달인 그룹
3. **MEV 미달** 근육 그룹
4. **가장 최근에 안 한** 근육 그룹

연속으로 같은 근육 그룹을 하지 않도록 최소 1일 휴식을 보장한다.

### Step 4. 운동 루틴 구성

추천된 근육 그룹에 맞는 운동을 `exercises.toml`에서 선택한다.

각 운동에 대해:
- **이전 기록 조회**: 최근 로그에서 같은 운동의 weight_kg, sets, reps를 찾는다
- **점진적 과부하 제안**: 이전 대비 무게 +2.5kg 또는 reps +1~2회 제안
- **주의사항 수집**: 이전 로그에서 해당 운동의 notes를 모두 수집

### Step 4-1. 운동 순서 결정 (필수)

선택한 운동을 다음 원칙에 따라 정렬한다. 컨디션이 가장 좋을 때 가장 중요한 운동을 배치하는 것이 핵심이다.

**순서 원칙**:

1. **활성화 / 워밍업** — 가벼운 코어 활성화 운동 (사이드 플랭크 + 밴드 등)
   - 이후 메인 운동의 안정성을 높이기 위함
   - 가벼운 강도여야 함 (본격적인 코어 운동은 후반부에)
2. **메인 compound** — 가장 무거운 다관절 운동, 목표 근육의 메인 자극
   - 컨디션 최고일 때 수행해야 점진적 과부하 가능
   - 예: 랫풀다운, 벤치프레스, 스쿼트, 데드리프트, 오버헤드 프레스
3. **보조 compound / isolation** — 같은 근육 그룹의 보조 운동
   - 예: 시티드 로우, 페이스풀, 인클라인 벤치프레스
4. **다른 근육 그룹의 isolation** — 별 근육 그룹이라 메인 운동에 큰 영향 X
   - 단, **무릎 컨트롤이 필요한 운동(싱글 레그 익스텐션 등 VMO)** 은 너무 뒤로 미루지 않음 (피로로 폼 무너짐)
5. **본격적인 코어** — 사이드 플랭크 (집중 세트), 케이블 우드찹, 플랭크
   - 메인 운동 전에 코어가 지치면 다른 운동의 안정성 떨어짐
6. **유산소 (러닝 등)** — 마지막
   - 근력 운동 전에 하면 글리코겐 소진으로 근력 ↓
   - 근력 운동 후 수행으로 지방 산화 효과도 ↑

**예외 처리**:

- 같은 운동(예: side_plank_band)을 활성화 목적과 본격 코어 목적으로 두 번 배치하지 않음 — 한 번만 배치
- 슈퍼세트(예: 바벨컬 + 라잉 트라이셉스)는 인접 배치
- `unstable = true` 기록이 있는 운동은 메인 운동 직후가 아닌 약간 후반부에 배치 (피로 누적된 상태에서 무게 증량 X, 자세 안정 우선)

**필수 출력**: Step 6에서 추천 결과를 출력할 때 위 순서대로 번호를 매겨 제시한다.

### Step 5. 예상 소요시간 계산

`profile.toml`의 `[preferences]`에서 `rest_sec_per_set`, `transition_sec`, `sec_per_rep`을 읽어 루틴 전체 소요시간을 추정한다.

**공식**:

```
총 시간(초) = Σ(운동별 시간) + (운동 수 - 1) × transition_sec

운동별 시간:
- reps 기반:       sets × (reps × sec_per_rep + rest_sec_per_set)
- duration 기반:   sets × (duration_sec + rest_sec_per_set)  (플랭크 등)
- cardio:          duration_sec  (러닝 등, 휴식 없음)
```

마지막 세트 이후 휴식도 포함되나, 운동 전환 시간과 상쇄되므로 단순화하여 위 공식을 사용한다.

**시간 예산 대응** (설정된 경우):
- 예상 시간이 `time_budget_min`을 초과하면, 아래 우선순위로 생략 후보를 제안한다:
  1. cardio (러닝 등) — 선택적 보강
  2. core (사이드 플랭크 등) — 보조 운동
  3. 해당 세션의 마지막 보조 운동
- 생략 후보는 사용자가 선택할 수 있도록 명시적으로 표시한다.

### Step 6. 추천 결과 출력

```
## 오늘의 운동 추천 (2026-04-16 수요일)

**추천 부위**: 등 + 이두
**이유**: 목표 근육(등) MAV 유지 필요, 이두 MEV 미달
**예상 소요시간**: 48분 (휴식 90초/세트 기준)

---

### 1. 랫풀다운
- 추천: 4세트 × 12회 × **52.5kg** (이전 50kg → +2.5kg)
- ⚠️ 주의: 마지막 세트 치팅 주의 (04/14 기록)

### 2. 시티드 로우
- 추천: 3세트 × 15회 × 42.5kg (이전 40kg → +2.5kg)

### 3. 페이스풀
- 추천: 3세트 × 15회 × 15kg
- 💡 트레이너 권장: 후면삼각근 보강 (04/10 리포트)

### 4. 바벨 컬
- 추천: 3세트 × 12회 × 20kg (이전 기록 없음 — 가벼운 무게로 시작)

---

📊 이 루틴 완료 시 주간 볼륨 예상:
- back: 18 → 28세트 (MAV 초과 — 다음 등 운동은 3일 후 권장)
- arms: 4 → 7세트

⏱️ 시간 구성:
- 메인 작업: 33분 / 휴식: 15분 / 전환: 4.5분
```
