# recipe Skill

사용자에게 레시피를 제공할 때 반드시 지켜야 하는 형식과 정밀도 기준이다.
이 스킬이 명시적으로 호출되지 않더라도, 레시피 텍스트를 생성하는 모든 상황(예: discord-message로 레시피 전송)에서 이 기준을 적용한다.

## 핵심 원칙

레시피는 "내가 다시 봐도 정확히 재현할 수 있어야 한다"는 기준으로 작성한다.
"적당히", "약간", "노릇하게" 같은 모호한 표현은 시각적 신호와 함께 정량적으로 보완한다.

## 필수 요소

### 1. 메타 정보

레시피 상단에 다음을 명시한다:
- 요리명
- **인분 수** (예: 1인분, 2인분)
- (선택) 총 소요 시간, 단백질/칼로리 등 영양 정보

### 2. 재료 (Ingredients)

**모든 재료**는 다음 두 가지를 함께 표기한다:

- **정확한 g 단위 중량** (필수)
- **계량하기 쉬운 단위** (큰술/작은술/공기/개/쪽 등 — 가능한 경우 병기)
- **손질 상태** (썰기 두께, 슬라이스 여부 등)

예시:
- ✅ `돼지목살 150g (1.5cm 두께 슬라이스)`
- ✅ `마늘 3쪽 (약 12g, 0.3cm 슬라이스)`
- ✅ `소금 1/2작은술 (약 2g)`
- ✅ `식용유 1큰술 (15ml)`
- ❌ `돼지목살 적당량` (모호함)
- ❌ `마늘 약간` (모호함)

### 3. 손질 단계 (Prep)

조리 시작 전 모든 손질을 별도 섹션으로 분리한다. 각 항목에 다음을 명시:

- **처리 방법** (씻기, 자르기 두께, 두드리기 등)
- **이유** (필요한 경우 — 예: "찬 고기 바로 구우면 속까지 안 익음")
- **소요 시간**

### 4. 조리 단계 (Cook)

각 단계마다 다음을 **모두** 포함한다:

| 항목 | 표기 방식 |
|------|----------|
| **시간** | 분/초 단위 정확히 (예: "2분", "30초", "1분 30초") |
| **불 세기** | 약불(3/10) / 중불(6/10) / 중강불(8/10) / 강불(10/10) 형태로 척도 병기 |
| **시각·청각 신호** | "치익 소리가 나야 적정", "흰자 가장자리가 살짝 바삭해질 때까지" 등 |
| **동작 방법** | "건드리지 말고", "집게로 세워서", "한 번도 뒤집지 말고" |

예시:
- ✅ `중강불(8/10)로 2분 예열 후 목살을 올린다. 닿자마자 "치익" 소리가 나야 적정 온도. 건드리지 말고 2분 굽기.`
- ❌ `센 불에서 적당히 굽는다.` (모호함)

### 5. 휴지 / 마무리

- 고기는 **휴지 시간** 명시 (육즙 가두기 목적).
- 동시 조리(병행) 가능한 단계는 명시 (예: "목살 휴지 중 병행").

### 6. 플레이팅 & 먹는 법

- 접시 위 배치 순서.
- 추천 조합/먹는 방법 (예: 쌈 조합, 소스 활용법).

## 실행 절차

### Step 1. 입력 확인

- 요리명과 인분 수를 파악한다. 인분 수가 명시되지 않으면 1인분 기준으로 작성한다.
- 사용자 보유 식재료 기반인 경우 `cooking/ingredients.md`를 참고할 수 있다.

### Step 2. 레시피 생성

위 템플릿과 필수 요소를 모두 충족하여 레시피를 작성한다.
모호한 표현이 들어갔다면 정량적 표현으로 교체한다.

### Step 3. 출력

- 단독 호출 시: 레시피를 응답으로 직접 출력한다.
- 다른 스킬(discord-message 등)에서 호출된 경우: 해당 스킬의 출력 채널로 전달한다.

## 주의사항

- **이 스킬은 레시피 생성 시 항상 적용된다.** 사용자가 "레시피 보내줘", "만드는 법 알려줘", "어떻게 해?" 등으로 요청할 때, 또는 다른 스킬이 레시피 컨텐츠를 필요로 할 때 이 형식을 따른다.
- 형식의 미적 요소(이모지, 구분선)보다 **정밀도(중량·시간·불세기)**가 우선이다. 플랫폼 제약(예: Discord 2000자 제한)이 있으면 미적 요소부터 줄이고 정밀도는 보존한다.
