# cooking

## products.toml

브랜드 가공식품의 영양성분을 키 기반으로 저장. logs에서 `products = ["키"]` 로 참조.
새 상품을 처음 기록할 때 영양정보가 주어지면 products.toml에 항목을 추가하고, logs에서는 키만 참조한다.

## logs 채점 기준

### effort (투입 노력) — 1~5

| 점수 | 기준                                                 |
| ---- | ---------------------------------------------------- |
| 1    | 데우기/끓이기만 (라면, 냉동식품 전자레인지)          |
| 2    | 재료 손질 없이 굽거나 볶기                           |
| 3    | 간단한 손질 + 조리 (썰기, 양념장 만들기 포함)        |
| 4    | 복잡한 손질 or 긴 조리시간 (마리네이드, 다단계 조리) |
| 5    | 복합 요리 (여러 메뉴 동시, 국+반찬 등)               |

### romantic — 1~3

| 점수 | 기준 |
| ---- | ---- |
| 1    | 대접 불가능 |
| 2    | 평상시에 가능 |
| 3    | 특별한 날에도 가능 |

### satisfaction (만족도/맛) — 1~5 — 절대적 맛 기준

| 점수 | 기준                                |
| ---- | ----------------------------------- |
| 5    | 외식 수준                           |
| 4    | 아내 또는 지인에게 대접 가능한 수준 |
| 3    | 혼자 먹기에 맛있는 수준             |
| 2    | 그럭저럭 먹을만한 수준              |
| 1    | 먹기 싫은 수준                      |
